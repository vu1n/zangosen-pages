### 2. ntoh*/hton* is a bad API
**[Read the Story](http://purplesyringa.moe/blog/ntoh-hton-is-a-bad-api/)**

#### Key Takeaways
- The current API for endian conversion in C (`ntohl` and `htonl`) conflates logical values with serialized data, leading to confusion among developers.
- A better design would separate logical types from their serialized formats for clarity and maintainability.
- The author advocates for treating endianness as a serialization format parameter rather than an intrinsic characteristic of data types.

#### Insightful Comments
- "This common misdesign could be rethought; the foundational issue is the association of endianness with the datatype." — [User](#womv0g)
- "Bit-level addressing would have likely reduced confusion over endianness." — [User](#xod17p)

#### Risks/Caveats
- Resistance to change in longstanding C APIs; significant backward compatibility issues could arise from redesigning these functions.

#### Who Should Care & Why
- C Language Users: Understanding better API design can guide effective coding practices and improve cross-platform development.

#### TL;DR
The APIs for endian conversion in C exhibit poor design, obscuring clarity and causing developer confusion.
